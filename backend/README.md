# backend

## 概要

沖縄台風ナビのバックエンド。実データ（JTWC / JMA）を優先しつつ、デモデータへのフォールバックも備えた設計。沖縄に特化した台風予測アプリのサーバーサイドです。

## 開発

```bash
bun install
bun run dev          # ホットリロード推奨
# または
bun run start
```

サーバーは `http://localhost:3000` で起動します。

## 主なエンドポイント

| エンドポイント                  | 説明                                      | 備考 |
|--------------------------------|-------------------------------------------|------|
| `GET /api/demo/state`          | アプリが主に使用する統合エンドポイント    | 内部で実データ優先 + デモフォールバック |
| `GET /api/typhoons/active`     | 実在の台風情報取得                        | JTWC優先、JMAフォールバック |
| `GET /api/typhoons/debug/sources` | 各データソースの取得状況確認           | 開発・デバッグ用 |
| `GET/POST/PATCH/DELETE /api/locations` | 保存場所のCRUD                       | デバイスID単位で分離（SQLite） |

## アーキテクチャのポイント

- **実データ優先**：`/api/typhoons/active` でJTWC/JMAから取得を試みる
- **クライアント側計算**：実データ取得時は、iOS側でも通知レベルに基づく到達時間を簡易計算
- **永続化**：保存場所はSQLite（`data/locations.db`）でデバイスごとに管理
- **通知レベル**：LOW / MEDIUM / HIGH / SEVERE をサポート（リスク計算に反映）

## データソース

- **JTWC**：風速半径（34/50/64kt）の現在・予報情報が比較的充実
- **JMA**：公式情報。風速半径の抽出は現在改善中

開発時はプロジェクトルートから `../run-dev.sh` を実行すると便利です。
