# 開発者向けデバッグガイド

## データソースの状況を確認する

### iOSアプリ側
- **地図左下**：現在のデータソース（実データ / デモデータ / 実データ取得失敗）が表示される
- **設定タブ**：詳細なデータソース状況とエラー内容を確認できる
- **場所タブ**：上部に実データ取得失敗時のバナーが表示される

### バックエンド側（開発時）
```bash
curl http://localhost:3000/api/typhoons/debug/sources
```

このエンドポイントで、各データソース（JTWC / JMA）の取得状況とエラー内容が確認できます。

## よくある状況と対応

### 実データが取れない
- ネットワークが不安定な場合や、台風シーズン外は取得に失敗しやすい
- その場合は自動でデモデータにフォールバックする
- アプリ上では「実データ取得失敗」としてオレンジの警告が表示される

### 再試行したいとき
- 地図・場所タブのエラーバナー内にある「再試行」ボタン
- 設定タブの「データを再読み込み」ボタン
- ツールバーの更新ボタン

### デバッグログ
- 実データ取得の成否はコンソールにログが出る（リリースビルドでは抑制されている）

## データベース（SQLite）

保存場所は `backend/data/locations.db` に保存されています。

```bash
# 確認例（sqlite3コマンドが必要）
sqlite3 backend/data/locations.db "SELECT * FROM locations;"
```

開発時は `data/` ディレクトリごと削除すればリセットできます。

## 実データ vs デモデータの違い

- **実データ**：JTWC/JMAから取得した実際の台風情報 + 風速半径
- **デモデータ**：架空の台風6号（Jangmi）を使った固定データ

アプリは実データを優先しますが、取得に失敗した場合は自動でデモに切り替わります。
