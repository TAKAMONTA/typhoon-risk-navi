# 沖縄台風ナビ - リリースノート（初回TestFlight / v0.9）

## バージョン情報
- **バージョン**: 0.9 (内部ビルド)
- **対象**: TestFlight 内部テスト配布用
- **リリース日**: 準備中

**本アプリは沖縄県に特化した台風予測・リスク可視化ツールです。**

---

## 主な新機能・改善

### 沖縄限定アプリへの方向転換
- アプリを「沖縄台風ナビ」としてリニューアル。
- 地図の初期表示を沖縄本島周辺（那覇周辺）に最適化。
- 沖縄県内の生活・旅行に役立つ台風リスク予測に特化。

### 精度モデル「動的減衰」の導入（最大の改善点）
- 台風が北上・弱体化するにつれて風速半径が縮小する現実的な傾向を、**台風自身の予報データから自動計算**して反映するようになりました。
- 計算要素：
  - 緯度進行（北上速度）
  - 最大風速の減少率
- 減衰率は通常約8%ですが、状況に応じて **4〜16%/日** の範囲で動的に調整されます。
- このモデルは**クライアント側の到達時間予測**と**バックエンドのリスク計算**の両方で完全に同期しています。

### マップ表示の強化
- 将来の予報ポイントにおける風速半径（34kt / 50kt / 64kt）も、動的減衰モデルを適用して時間とともに自然に縮小表示されるようになりました。
- ツールバーに現在の「精度モデル XX.X%」を表示し、どの予測モデルが使われているかを常に確認可能。

### 実データ優先 + 堅牢なフォールバック
- JTWC / 気象庁の実データを優先取得。
- 取得に失敗した場合も、明確なバナー表示＋ワンタップで再試行可能。
- デモデータへの自動フォールバック時も、リスク計算と地図表示は継続して利用できます。

### 保存場所・通知レベルの操作性向上
- 通知レベル（LOW / MEDIUM / HIGH / SEVERE）をリスト上で即座に変更可能。
- 優先度順ソート対応。
- HIGH / SEVERE の場所はマップ上でサイズとアイコンで視覚的に強調。

### その他の改善
- SQLiteによる永続化（アプリ再起動後も場所と通知レベルが保持）。
- 全体的なエラーハンドリングとローディング状態の表示を大幅に改善。

## 既知の制限（初回TestFlight時点）
- プッシュ通知はまだ実装されていません（将来のアップデートで対応予定）。
- 通知レベルは現在「視覚的な優先度とリスク計算」にのみ影響します。
- 実在の台風がない時期はデモデータで動作確認をお願いします。

## フィードバックのお願い
TestFlightでご利用いただき、以下の点についてフィードバックをいただけると大変助かります：

1. 実データ取得時の安定性（特に台風接近時）
2. 「精度モデル」の挙動が直感的かどうか
3. 保存場所の追加・編集・通知レベル変更の操作感
4. 地図上の情報の見やすさ（風速半径・進路・リスクマーカー）
